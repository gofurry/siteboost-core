# 恢复说明

## 恢复目标

`steam-accelerator restore` 的目标是恢复本项目修改过的系统状态，包括：

- System Proxy。
- PAC。
- hosts 标记区块。
- 与本项目相关的证书状态。

该命令计划在 `v0.3.0` 引入系统代理恢复，在 `v0.4.0` 引入 hosts 与证书相关恢复。

## 设计要求

- 每次修改系统状态前保存原值或 rollback 状态。
- rollback 状态应落在用户配置目录，而不是项目源码目录。
- hosts 只删除项目标记区块。
- 恢复失败时必须返回清晰错误。
- 恢复命令应能在服务未运行时独立执行。

## 手动恢复方向

具体平台命令会在实现对应模式后补齐。当前只保留设计方向：

- Windows：恢复 WinINet / 系统代理设置，清理 hosts 标记区块。
- macOS：恢复 networksetup 相关代理配置，清理 hosts 标记区块。
- Linux：首期不统一桌面代理恢复，只提供文档化指引。

## 验收标准

- 模拟崩溃后执行 `restore` 能恢复 PAC 或系统代理。
- Hosts 模式下执行 `restore` 能移除项目标记区块。
- 重复执行 `restore` 不应破坏用户原有配置。
